## 🎯 Key Visual Elements

### **1. Post Type Selection**
- **Two prominent tabs** for Goods/Services vs Events
- **Clear visual distinction** with icons and descriptions
- **Active state highlighting** for selected type

### **2. Content Input**
- **Large, prominent text area** similar to X/Twitter
- **Character counter** showing progress (1,247 / 5,000)
- **Placeholder text** to guide users
- **Focus states** with blue border and shadow

### **3. Image Upload**
- **Drag & drop zone** with dashed border
- **Preview thumbnails** for uploaded images
- **Add more button** for additional images
- **File type and size information**

### **4. Duration Selection**
- **Two radio button options** with clear descriptions
- **Quick preset buttons** for common durations
- **Custom date selection** with calendar widget
- **Visual feedback** for selected options

### **5. Tag System**
- **Primary tags** (Offer/Request) always visible for goods/services
- **Tags tab** that opens popup for additional categories
- **Selected tags display** showing current choices
- **Category icons** and descriptions for clarity

### **6. Action Buttons**
- **Save Draft** button for work in progress
- **Create Post** button for final submission
- **Disabled state** when form is invalid
- **Clear visual hierarchy** with primary action highlighted

## 🎨 Visual Design Features

✅ **Clean, modern interface** with consistent spacing  
✅ **Clear visual hierarchy** using typography and spacing  
✅ **Interactive elements** with hover and focus states  
✅ **Responsive layout** that works on all screen sizes  
✅ **Accessible design** with proper contrast and sizing  
✅ **Professional appearance** suitable for university use  

## 🔄 User Interaction Flow

1. **Select post type** → Tabs highlight and interface adapts
2. **Type content** → Character counter updates in real-time
3. **Add images** → Drag & drop or click to upload
4. **Set duration** → Choose preset or custom date
5. **Select tags** → Primary tags first, then additional categories
6. **Create post** → Form validation and submission

This visual mockup shows exactly what users will see and interact with, providing a clear understanding of the interface design and user experience without requiring any frontend development.
